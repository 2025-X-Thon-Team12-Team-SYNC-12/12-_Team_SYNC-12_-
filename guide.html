＜ Dr. Prompt 설명서 ＞

본 서비스는 사용자의 모호한 질문을 AI가 가장 잘 이해하는 명확하고 구조적인 프롬프트로 최적화합니다. 입력된 내용을 분석하여 핵심 요소를 강조하고 논리적으로 재구성하며, Gemini와 ChatGPT 각 모델의 특성에 딱 맞는 프롬프트를 자동으로 생성해 줍니다.

1) Gpt or Gemini 챗봇을 선택하고 대화하세요.

2) Gpt 모델 중 한 가지를 선택하세요.

a. Gpt-5 Instant: 빠른 대화, 간단한 QnA
b. Gpt-5 Thinking: 문제 풀이, 단계별 사고

3) Gemini 모델 중 하나를 선택하세요.

a. Gemini Flash: 빠른 답변
b. Gemini Thinking: 복잡한 주제

4) 초기 프롬프트를 입력하세요.

-> 프롬프트 설명서를 참고★

5) 챗봇과 함께 프롬프트를 발전시키세요!

＜ Prompt 구조화 ＞

1) GPT mentor

역할: GPT에게 페르소나를 부여하세요.

Ex) “네가 고양이라고 생각하고 참치에 대한 감상을 써줘.”

목표: 얻고자 하는 최종 산출물이 무엇인가요?

Ex) “ ~을 긍정적 또는 부정적으로 분류한 결과가 필요해.”

자료/맥락: 작업의 이유, 상황, 진도, 배경 등을 설명하세요.

Ex) “과제를 위해 이 문제를 풀고 있어. 이 부분을 모르겠어.”

제약조건: 작업에 적용되는 제한 사항이나 규칙을 명확히 하세요.

Ex) “딱딱한 공문서 말투보다는, 정중하지만 부드러운 해요체를 사용해줘.”

형식: 답변을 받고 싶은 형태(표, 리스트, 코드 등)를 정하세요.

Ex) “함수별 사용법과 장단점을 비교하는 표(Table)로 정리해줘.”

2) Gemini mentor

맥락 (Context): 작업의 배경, 목적, 현재 상황을 설명하세요.

Ex) "나는 코딩 초보자고, 이번 해커톤에서 파이썬을 처음 써봐."

역할 (Role): 제미나이에게 전문적인 페르소나를 부여하세요.

Ex) "너는 복잡한 개념을 아주 쉽게 설명해 주는 친절한 1타 강사야."

구체화 (Explicit Task): 해야 할 일과 제약 조건을 명확한 동사로 지시하세요.

Ex) "이 코드의 에러 원인을 분석하고, 주석을 달아서 수정된 코드를 보여줘."

형식 (Format): 답변을 받고 싶은 형태(표, 리스트, 코드 등)를 정하세요.

Ex) "수정 전/후를 비교할 수 있는 마크다운 표로 정리해 줘."

＜ Prompt 심화 이론 ＞

AI 모델이 최적의 결과를 생성하도록 프롬프트를 설계하고 최적화하는 기술 및 방법론

1) “Chain-Of-Thought” Prompting(CoT)

: 중간 과정을 단계별로 설명하면서 최종 답변에 도달하는 방식
=> Self-Consistency, Tree of Thoughts

2) “Zero-Shot” Prompting

: Few-Shot처럼 모델에 예시나 추가적 정보를 제공하지 않고,
질문에 직관적으로 답하게 하는 방식
: 모델이 주어진 지시문을 명확히 이해하지 못하거나 엉뚱한 답변을
할 가능성이 있다.

3) “Few-Shot” Prompting

: 적은 데이터만을 가지고 정확한 출력을 낼 수 있도록 하는 것
: 사용자가 원하는 질문에 대한 형식을 n개 입력한다. 이후 AI에게
질문을 입력하면 AI가 예시의 답변 형식에 맞춘 대답을 출력한다.

예)
[Prompt] 단어를 설명해줘.
[Ex] 희망이 뭐야? :어떤 일을 이루거나 하기를 바람.
또는 앞으로 잘 될 수 있는 가능성을 의미합니다.
[New] 절망이 뭐야?
[AI answer] : 바라볼 것이 없게 되어 모든 희망을 끊어버림. 또는 그런 상태를 의미합니다.
